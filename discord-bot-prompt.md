# Prompt: Discord bot — гүйлгээний мэдэгдэл + товчлуураар ангилах

## Зорилго

Голомт банкны гүйлгээ бүртгэгч системд **Discord bot** нэмэх. Шинэ гүйлгээ бүрд Discord суваг руу мэдэгдэл илгээнэ. Танигдаагүй гүйлгээнд **товчлуур/modal** дагалдаж, хэрэглэгч Discord-оос шууд ангилна (сайт руу орохгүй). Ангилсан зүйл learned override-д хадгалагдаж, dashboard-той ижил DB-д тусгагдана.

Зорилго: хэрэглэгч утаснаасаа Discord апп нээгээд, мэдэгдэл хараад, товч дараад ангилах — байнга сайт руу орохгүй.

## Контекст — одоо байгаа систем

- Listener (IMAP IDLE) → parse → categorize → API → SQLite → dashboard. EC2-д 24/7 ажиллаж байна.
- `categorize()` нь learned override → дүрэм шалгаж, танихгүй бол `null` → `pending_review`.
- 10 ангилал: Гадуур хооллолт, Хүнсний зүйл, Тээвэр, Орлого, Шилжүүлэг & гэр бүл, Захиалга & сервис, Боловсрол, Чөлөөт цаг / зугаа цэнгэл, Хувцас / гоо сайхан, Бусад.
- POS гүйлгээ (`is_pos`, BOM-төгсгөл) → газрын нэр асуудаг. Бусад (шилжүүлэг) → шалтгаан асуудаг.
- AI одоогоор off (кредитгүй).
- Dashboard хэвээр үлдэнэ (гэрт сайт). Discord болон dashboard ижил DB/API ашиглана.

## Хэрэглэгчийн оролцоо (бэлэн байх)
- Discord bot аль хэдийн үүсгэсэн. Хэрэгтэй: **Bot token**, **Channel ID** (мэдэгдэл ирэх суваг), bot server-т нэмэгдсэн байх.
- Developer Portal → Bot → Privileged Gateway Intents: шаардлагатай бол идэвхжүүл (доор заана).
- Эдгээр нууц утга (`DISCORD_BOT_TOKEN`, `DISCORD_CHANNEL_ID`) -г хэрэглэгч `.env`-д оруулна, агент log-д хэвлэхгүй.

## Технологи
- **discord.js** (Node.js) — bot framework. Listener/API-тай ижил Node 24 орчинд.
- Bot-ийг тусдаа процесс болгож pm2-д нэмэх (`bank-discord`), эсвэл одоо байгаа процесст нэгтгэх — тусдаа процесс илүү цэвэр (унтарвал бусдад нөлөөлөхгүй).

## Заавал хэрэгжүүлэх

### 1. Мэдэгдэл илгээх (бүх шинэ гүйлгээнд)
- Шинэ гүйлгээ DB-д орох/ангилагдах үед Discord суваг руу мэдэгдэл илгээнэ.
- **Хоёр төрөл:**
  - **Аль хэдийн ангилагдсан** (override/дүрмээр): зөвхөн мэдээллийн embed — огноо, тайлбар (friendly_name байвал тэр), дүн (зарлага улаан/орлого ногоон), ангилал ✓. Товч хэрэггүй.
  - **Танигдаагүй** (`pending_review`): ижил embed + **ангилах товчлуурууд** (доор).
- Embed-д: огноо, тайлбар/газрын нэр, дүн, төрөл, данс (сүүлийн орон), ангилал (эсвэл "?").

### 2. Товчлуураар ангилах (танигдаагүй гүйлгээ)
- Discord нэг мессежид дээд тал нь 25 товч (5 эгнээ × 5). 10 ангилал багтана.
- Танигдаагүй гүйлгээний мэдэгдэлд 10 ангиллын товчлуурууд (эгнээнд хуваасан).
- Хэрэглэгч товч дарахад:
  - **POS гүйлгээ (`is_pos`):** ангилал сонгосны дараа **modal (попап форм)** нээж "Газрын нэр?" (friendly_name) асууна. Жишээ: ShuluBOM → "Шулуун дун". Сонголтоор note.
  - **Шилжүүлэг (POS биш):** ангилал сонгосны дараа **modal** нээж "Шалтгаан? (хэнд, юунд)" асууна. Жишээ: HER-БАТСАЙХАН → "Ээжид сарын мөнгө".
  - Modal-ийн талбар сонголттой (хоосон орхиж болно — зөвхөн ангилал хадгалагдана).
- Баталгаажуулсны дараа:
  - Гүйлгээний `category` (+ `friendly_name`/`note`) шинэчлэгдэнэ (API-ийн `PATCH /api/transactions/:id/category` дуудна).
  - **Learned override хадгалагдана** (merchant_pattern → category + friendly_name). Тэр мерчантын ирээдүйн болон одоо байгаа бүх ижил гүйлгээнд хэрэглэх (`applyToAll`).
  - Discord мессежийг шинэчилж (edit) "✓ [Ангилал] болгож хадгаллаа" гэж харуулна, товчнуудыг устгана.

### 3. API-тай холбогдох (dashboard-той ижил)
- Bot нь шинэ endpoint биш, одоо байгаа API-г ашиглана:
  - Гүйлгээ ангилах: `PATCH /api/transactions/:id/category` (applyToAll-тэй).
  - Override нэмэх: `POST /api/overrides`.
- Bot API key-ээр authenticate хийнэ (`.env`-ийн `LISTENER_API_KEY` эсвэл тусдаа bot key).
- Discord болон dashboard ижил DB-д ажилладаг тул, Discord-оос ангилсан нь dashboard-д шууд тусна, эсрэгээр ч мөн.

### 4. Listener → Discord холбоос
- Listener гүйлгээ боловсруулсны дараа Discord мэдэгдэл илгээх. Хоёр арга:
  - **a)** Listener push хийхэд API талд "шинэ гүйлгээ" event үүсгэж, bot түүнийг сонсох (жишээ: API-аас bot руу webhook/internal call).
  - **b)** Bot нь DB-г тогтмол шалгаж (polling) шинэ `pending_review` гүйлгээ илрүүлэх.
  - **Санал:** (a) илүү real-time. Хэрэв хялбар байдлыг хүсвэл, API-д шинэ гүйлгээ орох үед bot-руу дотоод дуудлага хийх жижиг механизм. Polling (b) нь энгийн ч хоцролттой.
- Аль аргыг сонгосноо README-д тэмдэглэ.

### 5. Bot permissions / intents
- Bot-д хэрэгтэй: Send Messages, Embed Links, Read Message History.
- Button/Modal interaction-д Message Content Intent заавал биш (interaction нь тусдаа), гэхдээ slash command нэмбэл хэрэгтэй байж магадгүй — шаардлагатайг README-д зааж, хэрэглэгчид Developer Portal дээр идэвхжүүлэхийг сануул.
- Bot-ийг server-т урих OAuth2 URL (зөв permission-тай) -г README-д гаргаж өг.

### 6. Найдвартай байдал
- Bot тусдаа pm2 процесс (`bank-discord`), autorestart.
- Discord холболт тасрах үед discord.js автомат reconnect (default), нэмэлт хамгаалалт log.
- API дуудлага амжилтгүй (network) бол retry, алдааг log, гэхдээ bot унтрахгүй.
- Нэг гүйлгээнд давхар мэдэгдэл илгээхгүй (idempotency: илгээсэн messageId-г тэмдэглэх).

## Бүтэц
- `discord/bot.js` — bot үндсэн (мэдэгдэл, товч, modal handler)
- `discord/notify.js` — embed бүтээх, илгээх
- `discord/categories.js` — 10 ангилал → товчлуур mapping
- pm2 ecosystem-д `bank-discord` процесс нэмэх
- README: bot token авах, channel ID авах, intents, OAuth invite URL, .env тохиргоо

## .env нэмэлт
```
DISCORD_BOT_TOKEN=

# (API key аль хэдийн байгаа)
```

## Тест
- Шинэ ангилагдсан гүйлгээ → Discord-д мэдээллийн embed (товчгүй).
- Танигдаагүй гүйлгээ → embed + товчлуурууд.
- Товч дарах → POS бол газрын нэр modal, шилжүүлэг бол шалтгаан modal.
- Баталгаажуулсны дараа → DB шинэчлэгдэнэ, override хадгалагдана, Discord мессеж "✓ хадгаллаа" болж edit хийгдэнэ.
- Dashboard-аас шалгах: Discord-оос ангилсан гүйлгээ dashboard-д туссан эсэх.

## Чухал тэмдэглэл агентад
- Нууц утга (bot token) log-д хэвлэхгүй, хэрэглэгч .env-д оруулна.
- Bot тусдаа pm2 процесс — listener/API-д нөлөөлөхгүй.
- Discord болон dashboard ижил DB/API — хоёр тал ижил өгөгдөл харна.
- POS vs шилжүүлгийн ялгаа (газрын нэр vs шалтгаан) -ийг modal-д тусга.
- Idempotency: нэг гүйлгээнд давхар мэдэгдэл илгээхгүй.
- 10 ангилал товчлуурыг эгнээнд цэвэр хуваа (Discord 5×5 хязгаар).
- AI off хэвээр — Discord нь AI-аас хамаарахгүй, зүгээр хэрэглэгчийн товч/modal-аар ангилна.
